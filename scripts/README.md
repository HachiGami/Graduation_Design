# 分支嫁接脚本使用说明

## 功能
为现有的线性生产流程添加并行分支结构，用于演示CPM关键路径算法。

## 运行方式

### 标准模式（实际修改数据）
```bash
python -m scripts.graft_branches
```

### Dry Run模式（仅测试，不修改数据）
```bash
set DRY_RUN=true
python -m scripts.graft_branches
```

或Linux/Mac:
```bash
DRY_RUN=true python -m scripts.graft_branches
```

## 执行逻辑

1. **查找流程**：自动选择节点数 > 3 的前2个流程
2. **选择嫁接点**：在流程中随机选择两个间隔至少1跳的节点A和B
3. **智能匹配并行活动**：
   - 根据节点A和B的名称，匹配预设的业务规则库
   - 例如：`检验 -> 灌装` 匹配为"灌装线清洁消毒"
   - 如果没有匹配规则，使用通用"并行准备工作"
4. **创建旁路**：
   - 创建新Activity节点（具有真实业务意义）
   - 建立关系：`A -> New_Node -> B`
   - 耗时策略：
     - 准备类活动：耗时 = 原路径 × 0.6（非关键路径）
     - 质检类活动：随机设置（可能成为关键路径）
     - 其他活动：耗时 = 原路径 × 0.7（非关键路径）

## 业务规则示例

脚本内置了30+条生产业务规则，涵盖：

**生产流程**
- `接收 -> 包装`: 包材准备与检验
- `检验 -> 灌装`: 灌装线清洁消毒
- `消毒 -> 入库`: 冷库预冷与货位准备

**运输流程**
- `运输 -> 签收`: 单据准备与核对
- `装车 -> 卸货`: 客户通知与确认
- `运输 -> 卸货`: 卸货区预备

**销售流程**
- `订单 -> 拣货`: 库存预留与锁定
- `拣货 -> 配送`: 配送单生成与打印

**质检流程**
- `采集 -> 检测`: 检测设备校准
- `检测 -> 报告`: 历史数据对比分析

## 验证方式

运行脚本后，在前端页面刷新对应流程的图谱视图，应能看到：
- 新增的并行分支节点
- CPM关键路径高亮显示（如果新支路耗时更长，则成为新的关键路径）

## 数据库连接

脚本自动读取 `backend/app/config.py` 中的配置：
- MongoDB: `mongodb://localhost:27017`
- Neo4j: `bolt://localhost:7687`
