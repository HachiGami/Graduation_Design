# 依赖关系图交互说明

## 图布局

### 节点类型
- **Activity节点（活动）**：圆形，横向排列，表示生产流程中的各个活动
- **Resource节点（资源）**：矩形，位于活动下方，默认折叠
- **Personnel节点（人员）**：菱形，位于活动上方，默认折叠

### 关系类型
- **DEPENDS_ON（依赖关系）**：蓝色实线，粗线，连接活动节点
- **USES（使用资源）**：橙色虚线，细线，连接活动和资源
- **ASSIGNS（分配人员）**：绿色点线，细线，连接活动和人员

## 交互方式

### 左键点击
- **点击活动节点**：打开右侧抽屉，显示活动详情
  - 活动名称
  - 活动类型
  - 状态
  - 预计时长
  - 描述信息

### 右键点击
- **右键点击活动节点**：展开/折叠该活动的资源和人员节点
  - 首次右键：展开资源和人员节点
  - 再次右键：折叠资源和人员节点
  - 每个活动的资源和人员独立显示，不会重复

### 悬停提示
- **悬停活动节点**：显示操作提示
- **悬停资源/人员节点**：显示节点名称
- **悬停关系线**：显示关系详情（类型、数量、角色等）

### 画布操作
- **拖拽**：移动整个图
- **滚轮缩放**：放大/缩小
- **重置视图**：点击右上角"重置视图"按钮

## 资源和人员实例化

为了避免节点重复和混淆，系统采用实例化策略：

- **同一资源被多个活动使用**：为每个活动创建独立的资源节点实例
- **同一人员分配到多个活动**：为每个活动创建独立的人员节点实例
- **节点命名**：`{原始ID}_inst_{活动ID}`，确保唯一性

## 视觉设计

### 状态颜色
- **Activity状态**：
  - pending（待处理）：灰色
  - in_progress（进行中）：蓝色
  - completed（已完成）：绿色
  - paused（暂停）：橙色
  - cancelled（取消）：红色

- **Resource状态**：
  - available（可用）：绿色
  - in_use（使用中）：橙色
  - depleted（耗尽）：红色
  - reserved（预留）：蓝色

### 布局规则
- **横向布局**：活动节点从左到右排列
- **固定位置**：节点位置固定，不使用力导向布局
- **分层显示**：活动在中间，资源在下方，人员在上方
- **适度间距**：节点间距180px，避免拥挤

## 最佳实践

1. **查看流程**：默认只显示活动节点和依赖关系，清晰展示生产流程
2. **按需展开**：只展开感兴趣的活动，避免信息过载
3. **详情查看**：左键点击查看活动完整信息
4. **资源追踪**：右键展开查看活动具体使用的资源和人员
