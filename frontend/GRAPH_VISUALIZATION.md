# 依赖关系图可视化规则

## 设计原则

### 1. 节点分层与固定布局

**Activity节点（活动）**
- 按工序顺序分层，从左到右排列
- 层级划分：
  - L0 (reception): 奶源/接收
  - L1 (quality_control): 检验/消毒
  - L2 (processing): 加工/调配
  - L3 (packaging): 包装
  - L4 (storage): 仓储/物流
- 每层内部节点垂直排列，避免重叠

**Resource节点（资源）**
- 统一放置在图底部一排
- 不参与主流程布局
- 作为"附属"连接，按需显示

### 2. 关系显示控制

**默认显示**
- Activity节点（圆形）
- DEPENDS_ON关系（活动依赖，蓝色实线，较粗）

**按需显示**
- Resource节点（方形）
- USES关系（资源使用，橙色虚线，较细）
- 通过"显示资源"按钮切换

**点击交互**
- 点击Activity节点 → 右侧弹出资源使用面板
- 显示该活动使用的所有资源详情（数量、单位、阶段）

### 3. 标签显示优化

**节点标签**
- 显示节点名称，限制8个字符
- 超出部分显示省略号
- 悬停tooltip显示完整名称和详细信息

**关系标签**
- 默认不显示（避免满屏文字）
- 悬停tooltip显示关系详情：
  - DEPENDS_ON: 类型、时间约束、状态、描述
  - USES: 数量、单位、阶段

### 4. 节点去重机制

**后端保证**
- Neo4j查询使用DISTINCT去重
- Activity和Resource节点ID唯一
- 使用Set数据结构避免重复

**前端保证**
- 渲染前使用Map按ID去重
- 合并相同ID的节点数据
- 确保每个实体只显示一次

### 5. 边的视觉规范

**DEPENDS_ON（依赖关系）**
- 颜色：蓝色 (#409EFF)
- 样式：实线
- 宽度：2.5px
- 箭头：单向箭头
- 曲度：0.2（轻微弧线，避免重叠）

**USES（使用关系）**
- 颜色：橙色 (#E6A23C)
- 样式：虚线
- 宽度：1.5px
- 箭头：单向箭头
- 曲度：0.2

**布局规则**
- 所有箭头方向：从左到右（或从上到下）
- 使用固定布局（禁用力导向）
- 启用画布缩放和平移

## 交互功能

1. **显示/隐藏资源**：顶部按钮切换Resource节点和USES关系
2. **点击活动节点**：弹出右侧面板，展示资源使用详情
3. **悬停提示**：显示节点和关系的完整信息
4. **重置视图**：恢复初始视图和缩放比例
5. **邻接高亮**：点击节点高亮相邻的节点和关系

## 状态颜色映射

**Activity状态**
- pending（待处理）：灰色 (#909399)
- in_progress（进行中）：蓝色 (#409EFF)
- completed（已完成）：绿色 (#67C23A)
- paused（暂停）：橙色 (#E6A23C)
- cancelled（取消）：红色 (#F56C6C)

**Resource状态**
- available（可用）：绿色 (#67C23A)
- in_use（使用中）：橙色 (#E6A23C)
- depleted（耗尽）：红色 (#F56C6C)
- reserved（预留）：蓝色 (#409EFF)
